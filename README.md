## ✨ Features

- 📸 **Image to Recipe** — Upload a food photo and get a matching recipe generated by Gemini Vision AI
- ✍️ **Ingredients to Recipe** — Type what's in your fridge and get a tailored recipe instantly
- 🔄 **API Key Rotation** — Automatically rotates between 3 Gemini API keys if one hits its quota limit
- 🎨 **Premium UI** — Dark/light mode, glassmorphism panels, floating food animations
- ⚡ **Fast & Serverless** — Built on Next.js 15 App Router with edge-ready API routes
- 🛡️ **Secure** — API keys stored server-side, never exposed to the browser

## 🚀 Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) v18 or higher
- A [Google Gemini API Key](https://aistudio.google.com/app/apikey) (free tier available)

### 1. Clone the repository

```bash
git clone https://github.com/rajjimpact/ChefAi.git
cd ChefAi
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure your API keys

Create a `.env.local` file in the root of the project:

```env
GEMINI_API_KEY_1=your_first_gemini_api_key
GEMINI_API_KEY_2=your_second_gemini_api_key
GEMINI_API_KEY_3=your_third_gemini_api_key
```

> You can use just one key by filling only `GEMINI_API_KEY_1`. Keys 2 and 3 are used as fallbacks when the quota is exhausted.

### 4. Run the development server

```bash
npm run dev
```

Visit **http://localhost:3000/chef** to start generating recipes! 🎉

## 🔄 How API Key Rotation Works

The app includes a smart key rotation system (`app/lib/gemini.js`):

1. Every request first tries **Key 1**
2. If Key 1 returns a `429 Quota Exceeded` error, it instantly switches to **Key 2**
3. If Key 2 is also exhausted, it falls back to **Key 3**
4. If all keys are exhausted, a friendly error message is shown
5. Non-quota errors (wrong model, auth failures) fail immediately without rotating

This means your app stays alive even when individual free-tier keys hit their daily limits.

## 📦 Deployment

### Deploy to Vercel (recommended)

1. Push your code to GitHub
2. Import the repo on [vercel.com](https://vercel.com)
3. Add your environment variables (`GEMINI_API_KEY_1`, `GEMINI_API_KEY_2`, `GEMINI_API_KEY_3`) in the Vercel dashboard
4. Deploy!

> ⚠️ Never commit your `.env.local` file — it's excluded by `.gitignore`.
